<project_details>
    ModularAPITemplate is a .NET 10 template for building modular Web APIs with Clean Architecture. Each domain/module is self-contained with its own DbContext, endpoints, use cases (via MediatR), and DI registrations. Modules communicate via integration events (IEventBus — in-process by default, replaceable by RabbitMQ/Kafka/etc). A SharedKernel centralizes abstractions: Entity, IModule, BaseDbContext (auto-dispatches domain events), IEventBus, BaseWorker, Result<T>, IRequestContext (provides authenticated user info, extensible per module). The Host ties everything together. Developers focus on business rules, not infrastructure. Each module gets its own OpenAPI document (Scalar or Swagger UI). A CLI template (modularapi-module) scaffolds new modules. Architecture docs live in .ARCHITECTURE.md (English, source of truth) and .ARCHITECTURE.pt-BR.md (Portuguese).

    IMPORTANT: When using this for a real project, update this section with the actual project domain and requirements.
</project_details>
