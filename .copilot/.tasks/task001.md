# Task 001

## Description

The first task is to create the initial structure of the template. This includes setting up the necessary directories, files and projects for the project, including an example project/module/domain to demonstrate the structure and organization of the template. On the main project, include also a .ARCHITECTURE.md file with the architecture description and rules for the project.

## Steps to complete the task

1. Analyze the requirements and goals of the template to determine the necessary structure and organization.
2. Describe to the user the proposed structure and organization of the template, including the directories, files and projects that will be created.
3. Wait for the user's approval of the proposed structure and organization before proceeding with the implementation.
4. Create the necessary directories, files and projects for the template, including an example project/module/domain to demonstrate the structure and organization of the template.
5. Create a .ARCHITECTURE.md file on the main project with the architecture description and rules for the project.
6. Review the created structure and organization to ensure it meets the requirements and goals of the template.
7. Provide feedback to the user on the completed task and any next steps that may be necessary.

## Considerations

- The .ARCHITECTURE.md should be included in the template so that users can refer to it when creating their own projects using the template. It should provide clear and concise information about the architecture of the project, including any design patterns, principles, or guidelines that should be followed. This will help ensure consistency and maintainability across projects created with the template.
- The example project/module/domain should be designed to demonstrate the structure and organization of the template in a clear and concise manner. It should include relevant code and documentation to help users understand how to use the template effectively. This will help users get started quickly and easily when creating their own projects using the template.
- If possible, include the .copilot directory and the .tasks directory in the template, so that users can easily access and use the tasks when working on their projects. This will help ensure that users have access to the necessary tools and resources to complete their projects successfully.
